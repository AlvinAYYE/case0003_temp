# 對接問題清單

## 授權與環境

1. `x-access-token` 如何取得？token 有效期多久？
2. 正式環境 base URL 是什麼？是否需要 IP 白名單？
3. API 是否有 rate limit？超限錯誤與重試建議是什麼？

## 事件訂閱 / Webhook

1. callback 是否有簽名驗證？演算法與 header 欄位是什麼？
2. callback 失敗時會不會重試？重試次數與間隔是什麼？
3. 事件是否保序？是否可能重複推送？
4. `rawJson` 是否有每個 topic 的完整 schema 與範例？
5. 歷史事件只保留 3 個月，首次全量同步怎麼處理？

## 資料與業務口徑

1. 客戶、訂單、產品、員工的唯一鍵分別是什麼？
2. 刪除、停用、作廢的語意是否一致？
3. 財務資料的對帳口徑以哪個時間為準？

## 開發與維運

1. 是否有錯誤碼列表與固定 response 格式？
2. API 是否有版本升級與 breaking change 通知機制？
3. 正式上線前是否需要聯調驗收流程？
